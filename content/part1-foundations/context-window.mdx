# Context Window — ה-RAM של ה-AI

**Context Window** הוא מספר ה-Tokens המקסימלי שהמודל יכול "לראות" בו-זמנית. כל מה שמחוץ לחלון — לא קיים עבורו.

האנלוגיה הטובה ביותר: זה בדיוק כמו RAM במחשב.

## השוואה: RAM לעומת Context Window

| | מחשב (RAM) | AI (Context Window) |
|---|---|---|
| **גבול** | כמות הזיכרון הפיזי | 200K–1M Tokens (תלוי מודל) |
| **ככל שממלאים יותר** | פחות מקום לתוכנות | שיחה ארוכה יותר = פחות מקום |
| **כשמגיעים לגבול** | מחשב מאט | תשובות פחות טובות / שגיאות |
| **פתרון** | סוגרים תוכנות | מדחסים היסטוריה / מתחילים שיחה חדשה |

## מה ממלא את ה-Context Window?

1. **System Prompt** — ההוראות הראשוניות לכלי
2. **היסטוריית השיחה** — כל הודעה ותגובה מהעבר
3. **קבצים שנקראו** — כל קובץ שהכלי פתח
4. **פלט של כלים** — תוצאות ריצת פקודות
5. **התשובה הנוכחית** — מה שהמודל מייצר עכשיו

## ChatGPT לעומת כלי Agentic

זה ההבדל הקריטי שרוב המפתחים מפספסים:

- **ChatGPT** — רואה רק מה שאתם מדביקים. Context קטן, ידני, מוגבל.
- **Claude Code (Agentic)** — קורא קבצים, מריץ פקודות, רואה את הפרויקט כולו. Context מלא ואוטומטי.

  כלי Agentic לא "חכם יותר" מ-ChatGPT — הוא פשוט רואה יותר. Context הוא הכל.
</Callout>
