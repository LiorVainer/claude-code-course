# הלולאה האג׳נטית — איך Claude Code חושב

## זה לא קסם — זה קריאות לכלים

כשאתם נותנים ל-Claude Code משימה, הוא לא "חושב" בבת אחת ומוציא תשובה. הוא עובד בלולאה:

**חשוב → בחר כלי → הפעל → הסתכל על התוצאה → סיימת?**

אם לא סיים — חוזר לתחילת הלולאה.

## דוגמה: "תקן את הטסט הנכשל ב-auth.test.ts"

| שלב | פעולה | כלי |
|-----|--------|-----|
| 1 | קרא את קובץ הטסט | Read |
| 2 | קרא את קובץ ה-source | Read |
| 3 | זיהה את הבאג — null check חסר | (חשיבה) |
| 4 | ערוך את auth.ts שורה 42 | Edit |
| 5 | הרץ את הטסטים | Bash |
| 6 | הטסט עבר — סיים | (סיום) |

**6 קריאות לכלים. כל אחת גלויה. כל אחת מחכה לאישור שלכם.**
