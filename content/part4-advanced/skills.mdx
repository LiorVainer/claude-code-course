# Skills — תבניות פרומפט לשימוש חוזר

<Callout emoji="💡">
Skills הם תבניות פרומפט שנטענות רק כאשר צריך אותן — **lazy loading**. הם חוסכים tokens כי הם לא טעונים בזיכרון כל הזמן, אלא רק כשמשתמשים בהם.
</Callout>

## מה זה Skill?

Skill הוא בעצם **קיצור דרך לפרומפט** או **פיסת ידע קטנה** — קובץ Markdown פשוט שClaude טוען כשקוראים לו. במקום לכתוב את אותו הפרומפט שוב ושוב, מגדירים Skill פעם אחת ומשתמשים בו בפקודה פשוטה.

## איך Skill נראה מבפנים?

הנה דוגמה מקוצרת של Skill אמיתי — [`requesting-code-review`](https://github.com/obra/superpowers/blob/main/skills/requesting-code-review/code-reviewer.md) (24.2K התקנות):

```markdown filename="code-reviewer.md"
# Code Review Agent

You are reviewing code changes for production readiness.

**Your task:**
1. Review the implementation
2. Compare against requirements
3. Check code quality, architecture, testing
4. Categorize issues by severity

## Review Checklist

**Code Quality:**
- Clean separation of concerns?
- Proper error handling?
- Edge cases handled?

**Testing:**
- Tests actually test logic (not mocks)?
- All tests passing?

**Production Readiness:**
- Backward compatibility?
- Documentation complete?
```

זה הכל — קובץ Markdown עם הוראות. כש-Claude טוען את ה-Skill, הוא מקבל את הפרומפט הזה ישירות ל-Context ופועל לפיו. אין קוד, אין API — רק טקסט שמכוון את ה-AI.

## Lazy Loading — חיסכון ב-tokens

| | ללא Skills | עם Skills |
|---|---|---|
| פרומפטים | בזיכרון תמיד → בזבוז tokens | נטענים רק כשקוראים להם → חיסכון משמעותי |
