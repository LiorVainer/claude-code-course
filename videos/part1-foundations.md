## פרק: Tokens — האטומים של AI

> נתחיל מיחידת הבסיס — הדבר הכי קטן שה-AI מבין

### מה זה Token?

Token הוא חתיכת טקסט שהמודל למד לזהות. לא מילה שלמה, לא תו בודד — משהו באמצע. כלל אצבע: Token אחד ≈ 4 תווים באנגלית ≈ 0.75 מילים.

### דוגמאות Tokenization

```
"hello"              → 1 token

"unbelievable"       → 3 tokens
                       (un + believ + able)

"const x = 42;"      → 5 tokens
  const | x | = | 42 | ;
```

### קוד הוא יקר במיוחד

וזה מה שרוב המפתחים מפספסים — כל סימן בקוד שלכם הוא Token נפרד:

- כל סוגר, סוגריים ונקודה-פסיק = Token נפרד
- שרשראות שמות: getUserById → get + User + By + Id (4 tokens)
- רווחי הזחה = Tokens
- קובץ TypeScript של 100 שורות עולה 800–1,200 tokens

### ושפות לא-לטיניות? עוד יותר יקרות

עברית, ערבית, סינית — צורכות פי 3–4 יותר Tokens מאנגלית לאותו תוכן. זה משפיע על הערות, תיעוד, ו-Prompts.

### וכל Token הוא כסף

Output עולה פי 5 מ-Input. התגובות של ה-AI יקרות יותר מהבקשות שלכם.

חשבו על Tokens כמו חשבון החשמל של AI. לא צריך לספור כל ווט, אבל ברגע שמבינים שלהשאיר את כל האורות דלוקים עולה כסף — ההתנהגות משתנה.

## פרק: Context Window — ה-RAM של ה-AI

> עכשיו שהבנו את יחידת העלות — בואו נבין את יחידת המגבלה

### הזיכרון העובד של המודל

Context Window הוא מספר ה-Tokens המקסימלי שהמודל יכול "לראות" בבת אחת. חשבו על זה כמו RAM — הכל חייב להיכנס, אחרת דברים נדחפים החוצה.

### למה זה כמו RAM במחשב?

| מה שאתם מכירים מהמחשב | מה שקורה אצל ה-AI |
|------------------------|-------------------|
| יש גבול לזיכרון | 200K – 1M tokens (תלוי מודל) |
| ככל שפותחים יותר תוכנות — נגמר מקום | ככל שהשיחה ארוכה יותר — נגמר מקום |
| כשהזיכרון מלא — המחשב איטי | כשה-Context מלא — התשובות פחות טובות |
| סוגרים תוכנות כדי לפנות מקום | דוחסים היסטוריה כדי לפנות מקום |

### מה בעצם ממלא את ה-RAM הזה?

זה לא רק ה-Prompt שלכם — כל אלה חיים ב-Context Window בו-זמנית:

- System Prompt
- היסטוריית שיחה
- קבצים שנקראו
- פלט כלים
- התגובה הנוכחית

### למה זה משנה הכל

- ChatGPT רואה רק מה שהדבקתם — Context זעיר
- כלי אג׳נטי רואה את כל הפרויקט — Context מלא
- ניהול Context = ניהול איכות ה-AI
