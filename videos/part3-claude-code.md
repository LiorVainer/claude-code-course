## מה זה Claude Code?

> לפני שנצלול — בואו נבין מה הוא ומה הוא לא

### שלוש דרכים לדבר עם Claude

| דרך | מה זה | בשביל מי |
|-----|--------|----------|
| Claude AI | צ׳אט באתר או באפליקציה | כולם — שאלות ותשובות |
| API | קוד שמדבר עם קוד | מפתחים שמשלבים Claude באפליקציות |
| **Claude Code** | **סוכן AI שחי בטרמינל שלכם** | **מפתחים שרוצים שותף עבודה** |

### לא צ׳אט בוט — Agent

הוא קורא קבצים, מבין מבנה פרויקט, עורך קוד, מריץ פקודות, עושה git commit, ומריץ טסטים. הכל רץ על המחשב שלכם — הקוד לא עוזב את המחשב.

ההבדל מ-Cursor או Copilot? הם חיים בתוך עורך הקוד. Claude Code חי בטרמינל — לא תלוי בשום עורך ספציפי. יש גם אפליקציה, אבל הטרמינל הוא הבית שלו.

## הלולאה האג׳נטית

> אז איך הוא עושה את זה? כל פעולה שהסוכן מבצע — קריאת קובץ, עריכה, הרצת פקודה — היא קריאה לכלי. בואו נראה איך זה עובד.

### זה לא קסם — זה קריאות לכלים מובנים

כשנותנים ל-Claude משימה, הוא מריץ לולאה מובנית: חושב מה לעשות, בוחר כלי, מבצע, קורא את התוצאה, ומחליט אם סיים.

```
חשוב 🧠 → בחר כלי 🔧 → בצע ⚡ → צפה 👁 → סיים? ✓ → (חזרה לחשוב)
```

### בקשה אחת, 7 קריאות לכלים

```
❯ "fix the failing test in auth.test.ts"
  🧠 "I need to see the failing test"
  📖 Reading auth.test.ts...
  🧠 "I need to understand the function being tested"
  📖 Reading auth.ts...
  🧠 "Found it — missing null check at line 42"
  ✏️ Editing auth.ts — adding guard clause...
  💻 Running npm test auth.test.ts...
  ✓ All 12 tests pass!
```

כל פעולה נראית לכם. כל אחת דורשת את האישור שלכם. ברגע שמבינים את הלולאה — יודעים לכוון אותה.

## הכלים

> הלולאה מפעילה כלים — וכל פעולה היא קריאה מובנית שאתם יכולים לראות ולאשר

### ארגז הכלים של Claude Code

| כלי | תפקיד |
|-----|--------|
| Read 📖 | קריאת קבצים |
| Edit ✏️ | עריכה ממוקדת של קוד קיים |
| Write 📝 | יצירת קבצים חדשים |
| Bash 💻 | הרצת פקודות shell — טסטים, build, git |
| Grep 🔍 | חיפוש תוכן בקבצים |

### למה חשוב להבין את הכלים?

- כשנשאלים "Can I edit auth.ts?" — זה Edit tool. יודעים בדיוק מה יקרה
- יכולים לחזות: "Claude כנראה יצטרך Read ואז Edit"
- אם Claude תקוע בלולאה — רואים אילו כלים הוא חוזר עליהם
- פרומפט טוב יותר: "read auth.ts and fix the null check at line 42" — מכוון את בחירת הכלי

## הרשאות ואבטחה

> אם הוא מריץ פקודות לבד, מי שולט? אתם.

### אבטחה כברירת מחדל

Claude Code מתחיל עם אבטחה מקסימלית. כל פעולה דורשת אישור מפורש. אתם רואים בדיוק מה Claude רוצה לעשות לפני שזה קורה.

```
Claude wants to: Edit auth.ts

[Allow once] [Allow for session] [Allow always] [Deny]
```

### הגדרת פקודות מותרות מראש

אמרו ל-Claude אילו פקודות תמיד בסדר ואילו חסומות — דרך settings.json:

```json
{
  "permissions": {
    "allow": [
      "Bash(git *)",
      "Bash(npm test*)"
    ],
    "deny": [
      "Bash(rm -rf *)",
      "Bash(*DROP TABLE*)"
    ]
  }
}
```

### בנו אמון בהדרגה

```
יום 1: אשרו הכל ידנית 🔒
שבוע 1: הרשאות Session 🔓
שבוע 2: Bash מותר ב-settings ✅
חודש 1: Allow always 🚀
אח״כ: אוטומטי מלא ⚡
```

## CLAUDE.md

> ספר ה-Onboarding של הפרויקט — מה הוא צריך לדעת ביום הראשון?

### מה לשים בו

הקובץ נטען אוטומטית ל-Context בתחילת כל סשן. שימו בו את הטכנולוגיות, הכללים והפקודות של הפרויקט:

```markdown
# MyApp
## Stack
- TypeScript strict mode, no any
- React 19 + TanStack Router
## Rules
- Named exports only
- Run tsc after every change
## Commands
- npm test / npm run build
```

### היררכיה וכללי אצבע

- שמרו מתחת ל-200 שורות — הקובץ נטען לכל אינטראקציה ועולה Tokens
- הוראות מפורטות שייכות ל-Skills (טעינה לפי דרישה, נלמד בחלק ד׳)
- שלוש רמות: גלובלי ← פרויקט ← תיקייה. הספציפי דורס את הכללי

| רמה | נתיב |
|-----|------|
| גלובלי — כל הפרויקטים | ~/.claude/CLAUDE.md |
| פרויקט — כללים ספציפיים | /project/CLAUDE.md |
| תיקייה — לתת-מערכות | /project/src/CLAUDE.md |

### Auto Memory — הוא לומד לבד

Claude כותב לעצמו הערות תוך כדי עבודה. מגלה שהפרויקט משתמש ב-npm ולא ב-pnpm? שומר. בסשן הבא הוא כבר יודע. הפקודה memory/ מראה מה הוא שמר.

### נקודה חשובה: Claude Code הוא Stateless

בכל הודעה שאתם שולחים — הוא מעבד מחדש את כל היסטוריית השיחה. ככל שהשיחה ארוכה יותר, יותר Tokens נשרפים. הפקודה compact/ מסכמת את ההיסטוריה ומפנה מקום. אל תחכו שייגמר — עשו את זה כל כמה משימות.

### כלל 90 / 9 / 1

| מודל | מתי | אחוז שימוש |
|------|-----|------------|
| **Sonnet 🎯** | **פיצ׳רים, bug fixes, refactoring, code review** | **90%** |
